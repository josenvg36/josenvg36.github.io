My Experience Creating a Website with Flask, Fly.io, and GCP

Sharing my journey of web development with Flask and deploying on Fly.io and Google Cloud Platform.

Starting with Flask

I chose Flask as the framework for my website due to its simplicity and flexibility. Flask allows you to build web applications quickly without the overhead of more extensive frameworks. Here’s a step-by-step breakdown of my process:

Setting Up Flask:

I started by installing Flask and other necessary dependencies using pip:

pip install Flask

I created a simple Flask application to serve as the foundation for my website. Here’s a basic example of the app.py file I used:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Local Testing:

Running the application locally was straightforward. I used the command:

python app.py

The website ran smoothly on my local machine, which gave me the confidence to move forward with deploying it to a cloud server.

Attempting to Use Fly.io

My initial choice for deployment was Fly.io, a cloud platform known for its simplicity and developer-friendly features.

Setting Up Fly.io:

I followed the official Fly.io documentation to set up my environment and deploy the Flask app. The process involved installing Flyctl, Fly.io's command-line tool, and creating a new Fly.io application:

flyctl launch

During this setup, I encountered issues with generating the necessary Docker files for my Flask app.

Encountering Issues:

Despite following the steps meticulously, I ran into errors that prevented the successful creation of Docker images. These errors hindered my ability to link my Flask application with the Fly.io server. The error messages were vague, making troubleshooting challenging. After several attempts, I decided to explore other options.

Switching to Google Cloud Platform (GCP)

Given the challenges with Fly.io, I turned to Google Cloud Platform (GCP), which offers robust infrastructure and extensive documentation.

Creating a Virtual Machine (VM):

I started by setting up a VM instance on GCP. Initially, I chose Debian as the operating system but soon switched to Ubuntu for its familiarity and ease of use. Creating a new VM with Ubuntu was straightforward using the GCP Console.

Setting Up the Environment:

On the new VM, I installed the necessary packages for running Flask:

sudo apt update
sudo apt install python3-pip
pip3 install Flask

I transferred my Flask application files to the VM using SCP (Secure Copy Protocol).

Deploying the Flask Application:

After configuring the VM, I ran my Flask application. To ensure it was accessible over the internet, I configured the firewall rules to allow HTTP traffic. Running the Flask application on the VM was a success. My website was live, and I could access it via the external IP provided by GCP.

Conclusion

This journey taught me valuable lessons about deploying web applications. Here are my key takeaways:

  • Flask is a powerful and easy-to-use framework for web development.
  • Fly.io, while promising, had some deployment challenges that I couldn’t overcome.
  • Google Cloud Platform proved to be a reliable and versatile solution for hosting my Flask application.

Overall, using GCP for deploying my Flask app was a great success, and I look forward to building more projects with this setup. From now on, I’ll confidently use Flask in conjunction with Google Cloud services to host my web applications.
